# Ejercicios HTML bases (con IA)

Referencia de continuidad: `jp/bitacora/CLASE1.md`.
Tema: HTML bases.
Nivel: inicial.
Resultado esperado: crear páginas creativas usando la mayor cantidad posible de etiquetas HTML básicas vistas en clase.

## Ejercicio 1 — Página "Mi día ideal"
**Objetivo técnico:** usar estructura semántica básica y variedad de etiquetas de contenido.

**Prompt inicial recomendado:**
"Crea una página HTML llamada 'Mi día ideal' usando `header`, `main`, `section` y `footer`, con títulos, párrafos, listas, enlaces e imágenes. Quiero una estructura clara y legible para nivel inicial."

**Cómo iterar el prompt:**
- Iteración 1: "Añade una navegación interna con anclas a cada sección".
- Iteración 2: "Reorganiza para mejorar jerarquía de títulos y semántica".

**Checklist de validación:**
- [ ] Usa etiquetas semánticas básicas.
- [ ] Tiene al menos 3 secciones distintas.
- [ ] Incluye `h1`, `h2`, `p`, `ul`/`ol`, `a`, `img`.

**Errores frecuentes a evitar:**
- Repetir varios `h1` sin sentido.
- Poner imágenes sin `alt`.
- Usar solo `div` para todo.

## Ejercicio 2 — Página "Guía rápida de hobbies"
**Objetivo técnico:** resolver un problema informativo con estructura clara y enlaces útiles.

**Prompt inicial recomendado:**
"Genera una página HTML de guía de hobbies para principiantes. Incluye introducción, categorías en listas, tabla simple de tiempo/material y enlaces externos. Mantén semántica básica y código limpio."

**Cómo iterar el prompt:**
- Iteración 1: "Añade una sección de preguntas frecuentes con enlaces internos".
- Iteración 2: "Simplifica el contenido para lectura rápida en 2 minutos".

**Checklist de validación:**
- [ ] Contenido organizado por secciones.
- [ ] Incluye lista y tabla simple.
- [ ] Los enlaces funcionan y son relevantes.

**Errores frecuentes a evitar:**
- Tabla sin encabezados.
- Enlaces sin texto descriptivo.
- Estructura visual confusa.

---

## Ejercicio 3 — Formulario creativo de registro
**Objetivo técnico:** diseñar un formulario semántico con campos básicos y validación nativa.

**Prompt inicial recomendado:**
"Crea una página HTML con un formulario de registro para un club creativo. Debe incluir nombre, correo, edad, interés principal, mensaje y botón enviar. Usa `label` correctamente, `type` adecuados y campos requeridos donde tenga sentido."

**Cómo iterar el prompt:**
- Iteración 1: "Añade `fieldset` y `legend` para agrupar campos".
- Iteración 2: "Revisa accesibilidad básica y mejora etiquetas".

**Checklist de validación:**
- [ ] Cada campo tiene su `label` asociado.
- [ ] Se usa `type="email"` para correo.
- [ ] Campos clave marcados como `required`.
- [ ] Formulario legible y ordenado.

**Errores frecuentes a evitar:**
- `label` sin `for` o sin `id` en input.
- Campos obligatorios mal definidos.
- Mezclar estructura sin semántica.

## Ejercicio 4 — Reto libre: mini web temática
**Objetivo técnico:** integrar todas las etiquetas básicas aprendidas para resolver un caso creativo.

**Prompt inicial recomendado:**
"Propón y genera una mini web temática (por ejemplo viajes, videojuegos, música o cocina) usando el máximo de etiquetas HTML básicas vistas en clase. Quiero estructura semántica, enlaces, listas, imagen, tabla simple y formulario de contacto."

**Cómo iterar el prompt:**
- Iteración 1: "Haz el contenido más claro para nivel inicial".
- Iteración 2: "Refactoriza el HTML para que sea más semántico y mantenible".

**Checklist de validación:**
- [ ] Integra en una sola web los bloques principales de HTML básico.
- [ ] Se entiende la intención de cada sección.
- [ ] El código final fue revisado y corregido manualmente.

**Errores frecuentes a evitar:**
- Aceptar la primera respuesta de IA sin revisar.
- Perder coherencia entre secciones.
- Código repetitivo innecesario.

## Reflexión final (obligatoria)
Responder en 4 líneas:
1. ¿Qué te generó la IA que realmente te sirvió?
2. ¿Qué tuviste que corregir manualmente y por qué?
3. ¿Qué etiquetas HTML reforzaste mejor hoy?
4. ¿Qué mejorarías en tu próximo prompt?
