# Ejercicio de refuerzo: Tarjeta de perfil de Senior Cat

## Objetivo
Reforzar fundamentos de HTML y CSS con un ejercicio corto:
- estructura básica en HTML,
- estilos esenciales en CSS (espaciado, tipografía y caja visual).

**Nivel:** inicial  
**Duración estimada:** 25-35 minutos

---

## Consigna
Crear una mini página con:
- un título principal,
- un subtítulo corto,
- una imagen (puede ser de Senior Cat),
- un párrafo de presentación,
- una lista con 3 habilidades.

Luego aplicar estilos para que se vea como una “tarjeta” centrada:
- fondo claro,
- márgenes y padding consistentes,
- bordes redondeados,
- texto legible y alineado.

---

## Requisitos mínimos
- Usar etiquetas semánticas básicas: `header`, `main`, `section`, `footer`.
- Separar HTML y CSS en archivos distintos.
- Aplicar al menos 6 propiedades CSS:
  - `margin`
  - `padding`
  - `color`
  - `background-color`
  - `border-radius`
  - `font-family`
- Mantener jerarquía correcta de títulos (`h1` y `h2`).

---

## Pistas (sin solución)
- Piensa la tarjeta como un bloque dentro de `main`.
- Primero consigue orden visual general, después ajusta detalles.
- Si algo se ve “pegado”, revisa `margin` y `padding`.

---

## Criterio de finalización
- El contenido se entiende en menos de 5 segundos.
- La tarjeta tiene separación interna clara.
- No hay estilos en línea (todo en CSS externo).
- Se ve bien al reducir el ancho de la ventana.

---

## Mini guía de trabajo en clase
- 5 min: estructura HTML.
- 10 min: estilos base en CSS.
- 10 min: mejora visual.
- 5 min: revisión final.

---

## Cierre
Construyendo el futuro del desarrollo web, un ladrillo a la vez.  
Senior Cat recomienda enfocarse en claridad antes que en decoración.

# Enunciado 2 (nuevo): Tarjeta modo DNI de Senior Cat

## Objetivo
Tomar la tarjeta anterior y transformarla en un formato más compacto tipo DNI, manteniendo fundamentos de HTML y CSS.

**Nivel:** inicial-intermedio  
**Duración estimada:** 30-40 minutos

---

## Consigna
Partiendo del ejercicio anterior, adapta el diseño para que se vea como una tarjeta tipo DNI:
- bloque horizontal más ancho que alto,
- imagen de Senior Cat a un lado,
- datos principales al otro lado,
- pie de tarjeta discreto en la parte inferior.

---

## Requisitos mínimos
- Mantener estructura semántica: `header`, `main`, `section`, `footer`.
- Mantener CSS externo (sin estilos en línea).
- Usar `display: flex` para ordenar la tarjeta.
- Mantener espaciado consistente con `margin` y `padding`.
- Mantener legibilidad: jerarquía clara entre `h1`, `h2`, texto y lista.

---

## Pistas (sin solución)
- Haz primero el bloque principal de la tarjeta y luego ajusta detalles.
- Usa un ancho máximo para que no se estire demasiado en pantallas grandes.
- Si el footer no queda abajo, revisa layout vertical de página (`body` y `main`).

---

## Criterio de finalización
- Se percibe visualmente como tarjeta tipo DNI.
- Foto y datos están alineados y no se pisan.
- El contenido sigue siendo legible en ancho reducido.
- No hay reglas CSS redundantes o contradictorias.

---

## Mini guía de trabajo en clase
- 10 min: ajustar estructura visual tipo DNI.
- 10 min: layout con flex y alineaciones.
- 10 min: espaciado y ajustes finos.
- 5-10 min: revisión final con checklist.

---

## Cierre del enunciado 2
Mismo enfoque del grupo Ladrillos: construir desde una base clara y mejorar por iteraciones, un ladrillo a la vez con Senior Cat.

# Práctica extra sin IA: HTML + CSS (fácil, 30 minutos)

## Objetivo de la sesión
Construir una página simple y ordenada, reforzando fundamentos de estructura HTML y estilos CSS básicos.

**Nivel:** inicial  
**Duración total:** 30 minutos

Construyendo el futuro del desarrollo web, un ladrillo a la vez, con Senior Cat como guía.

---

## Ejercicio 1 (10 min): Estructura base

**Objetivo**
Crear una página con estructura semántica clara.

**Requisitos mínimos**
- Usar `header`, `main`, `section` y `footer`.
- Incluir un `h1`, un `h2`, un párrafo y una lista `ul` con 3 elementos.
- Guardar el HTML sin estilos en línea.

**Pistas (sin solución)**
- Piensa cada etiqueta como un ladrillo con una función concreta.
- Revisa que solo haya un `h1`.

**Criterio de finalización**
- La estructura se entiende al leer solo el HTML.

---

## Ejercicio 2 (10 min): Estilos básicos

**Objetivo**
Aplicar estilos mínimos para mejorar legibilidad.

**Requisitos mínimos**
- Enlazar un archivo CSS externo.
- Aplicar `font-family`, `color`, `background-color`, `margin` y `padding`.
- Dar estilo a enlaces con `a`, `a:hover` y `a:focus` sin cambiar tamaño de fuente.

**Pistas (sin solución)**
- Evita cambios bruscos en `hover`.
- En `focus`, usa `outline` para accesibilidad.

**Criterio de finalización**
- El texto se lee bien y los enlaces muestran estados claros.

---

## Ejercicio 3 (10 min): Mini tarjeta de Senior Cat

**Objetivo**
Convertir el contenido en una tarjeta simple y limpia.

**Requisitos mínimos**
- Crear un contenedor con clase (por ejemplo, `.tarjeta`).
- Usar `max-width`, `border`, `border-radius` y `box-sizing`.
- Dejar separación interna consistente entre bloques.

**Pistas (sin solución)**
- Primero ordena el contenedor general; luego ajusta detalles.
- Si algo se ve pegado, revisa `margin` y `padding`.

**Criterio de finalización**
- La tarjeta se distingue visualmente y mantiene buena lectura.

---

## Mini guía de trabajo (orden recomendado)
- 0-10 min: Ejercicio 1
- 10-20 min: Ejercicio 2
- 20-30 min: Ejercicio 3

## Checklist final
- [ ] HTML semántico básico completo.
- [ ] CSS externo enlazado correctamente.
- [ ] Enlaces con `hover` y `focus` legibles.
- [ ] Página clara y ordenada.

# Enunciado 3 (alternativo, sin tarjeta): Menú de cafetería de Senior Cat

## Objetivo
Practicar HTML y CSS básico construyendo una página tipo menú (desayuno, bebidas y postres), con estructura clara y estilos simples.

**Nivel:** inicial  
**Duración estimada:** 30 minutos

---

## Consigna
Crear una página de “Cafetería Senior Cat” que incluya:
- un encabezado con nombre del local y frase corta,
- 3 secciones de menú (`Desayuno`, `Bebidas`, `Postres`),
- en cada sección una lista con al menos 3 productos,
- un pie de página con horario o contacto.

---

## Requisitos mínimos
- Usar etiquetas semánticas: `header`, `main`, `section`, `footer`.
- Usar `ul` y `li` para los productos.
- Separar HTML y CSS en archivos diferentes.
- Aplicar como mínimo estas propiedades CSS:
  - `font-family`
  - `color`
  - `background-color`
  - `margin`
  - `padding`
  - `border`
  - `border-radius`

---

## Pistas (sin solución)
- Piensa cada sección como un ladrillo del menú completo.
- Empieza por estructura HTML y luego da estilo por bloques.
- Si todo se ve junto, revisa espaciado (`margin` y `padding`).

---

## Criterio de finalización
- Se distinguen claramente las 3 secciones del menú.
- El texto es legible y el contraste es correcto.
- No hay estilos en línea.
- La página mantiene orden visual al cambiar el ancho de la ventana.

---

## Mini guía de trabajo (30 min)
- 0-10 min: estructura HTML completa.
- 10-20 min: estilos base globales (tipografía, colores, espaciado).
- 20-30 min: acabado de secciones y revisión final.

## Cierre
En Ladrillos, avanzamos por bloques: base primero, detalle después. Senior Cat aprueba esta práctica porque fortalece fundamentos reales sin depender de IA.
