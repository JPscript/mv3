# CRUD de Recetas con API NestJS

## Enunciado

Senior Cat y el grupo Ladrillos te invitan a construir una página web que se conecte con tu API de recetas (NestJS) y permita realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) sobre las recetas.

### Objetivo
Crear una interfaz web sencilla, con botones para cada acción CRUD:
- **Create:** Al hacer clic, te lleva a una página/formulario para crear una nueva receta y guardarla en la API.
- **Read:** Al hacer clic, te lleva a una página donde puedes ver todas las recetas almacenadas.
- **Update:** Al hacer clic, te lleva a una página para modificar una receta existente.
- **Delete:** Al hacer clic, te lleva a una página para eliminar una receta.

Cada acción debe tener su propio archivo HTML, CSS y JS. El diseño debe ser claro, con instrucciones visibles y botones bien identificados.

### Requisitos mínimos
- Usar solo HTML, CSS y JS puro (sin frameworks).
- Conectar con la API usando fetch.
- Separar cada acción en su propio archivo.
- Mostrar mensajes claros de éxito o error.

### Guía de inicio
1. Crea la carpeta `CLASE5` dentro de `jose/sin-ia/HTML`.
2. Dentro de esa carpeta, crea:
   - `README.md` (este archivo)
   - `index.html` (menú principal con los botones CRUD)
   - `create.html`, `read.html`, `update.html`, `delete.html` (una por acción)
   - Archivos CSS y JS para cada página.
3. El menú principal debe tener los 4 botones, cada uno enlazando a la página correspondiente.

### Criterio de finalización
- Cada acción debe funcionar y comunicarse con la API.
- El menú debe permitir navegar entre las páginas.
- El código debe estar ordenado y comentado.

### Metáfora de construcción
Cada página y botón es un ladrillo en tu muro de habilidades. Senior Cat supervisa y te anima a avanzar paso a paso, asegurando que cada ladrillo esté bien colocado.
