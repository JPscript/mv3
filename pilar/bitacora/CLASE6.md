## Resultados de aprendizaje de la sesión

- Introducción a JavaScript aplicado al frontend: Comprendimos cómo JavaScript permite transformar una página estática en una interfaz dinámica capaz de mostrar datos reales provenientes de internet.

- Consumo de APIs con fetch: Aprendimos a realizar peticiones a una API pública para obtener datos externos y utilizarlos dentro de una página web.

- Manejo básico de asincronía: Introducción al uso de async y await para trabajar con respuestas de red de forma más clara y legible.

- Transformación de datos con map: Utilizamos métodos de arrays para convertir los datos recibidos en estructuras HTML reutilizables.

## Conocimientos adquiridos en con los ejercicios de la sesión

Durante esta sesión dimos el paso de páginas web estáticas a interfaces dinámicas. Introdujimos JavaScript como herramienta fundamental para interactuar con datos externos y modificar el contenido de la página en tiempo real.

Aprendimos a realizar peticiones a una API pública utilizando fetch, entendiendo que una API es una fuente de datos que permite a diferentes aplicaciones comunicarse entre sí a través de internet. Analizamos la estructura de las respuestas en formato JSON y aprendimos a identificar qué campos son útiles para nuestro proyecto.

Posteriormente transformamos esos datos utilizando map, lo que nos permitió generar automáticamente múltiples tarjetas HTML a partir de un conjunto de datos. Este proceso nos ayudó a comprender cómo los datos se pueden convertir en elementos visuales dentro de una interfaz.

Finalmente, integramos todo el flujo: solicitar datos, procesarlos y renderizarlos en pantalla dentro del DOM. Esto nos permitió construir una página que muestra información real proveniente de una API, incluyendo imagen, título y descripción para cada tarjeta generada dinámicamente.

Durante la práctica surgieron algunas dificultades técnicas habituales, especialmente relacionadas con la correcta conexión a la API y con la identificación de los campos correctos del JSON para poder mostrar los datos en pantalla. Estas situaciones sirvieron para reforzar el uso de herramientas como console.log y la inspección del JSON para entender mejor cómo trabajar con datos externos.
