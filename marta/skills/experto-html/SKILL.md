# Experto HTML

## Cuándo usar esta skill
- Cuando quieras crear o mejorar archivos HTML de ejercicios.
- Cuando necesites revisar semántica y estructura del documento.
- Cuando quieras validar formularios y accesibilidad básica.

## Procedimiento
1. Identifica objetivo del ejercicio y carpeta correcta (`con-ia` o `sin-ia`).
2. Crea o revisa estructura semántica (`header`, `main`, `section`, `article`, `footer`).
3. Verifica formularios (`label` asociado, `name`, `type`, `required` cuando aplique).
4. Revisa enlaces e imágenes (`href` válido, `alt` descriptivo).
5. Entrega cambios mínimos y explicación breve en español.

## Checklist rápido
- Documento con estructura clara y jerarquía de títulos (`h1` a `h3` según necesidad).
- Uso de etiquetas semánticas en lugar de `div` genérico cuando aplica.
- Formularios con etiquetas accesibles.
- Sin agregar librerías o frameworks no solicitados.

## Entrada sugerida
- Tema del ejercicio.
- Ruta objetivo.
- Si el trabajo es `con-ia` o `sin-ia`.

## Salida esperada
- HTML funcional, simple y consistente con el curso.
- Resumen corto de qué se cambió y por qué.
